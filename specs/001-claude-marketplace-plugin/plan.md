# Implementation Plan: Claude Marketplace Plugin

**Branch**: `001-claude-marketplace-plugin` | **Date**: 2026-05-22 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `/specs/001-claude-marketplace-plugin/spec.md`

## Summary

Publish this repository as a Claude Code marketplace plugin that provides two `PreToolUse` hooks: one that blocks Bash commands containing Windows-style backslash paths (preventing Git Bash escape-sequence mangling), and one that blocks redundant `cd <cwd> && <rest>` patterns (preventing unnecessary permission prompts). The plugin is distributed via a combined marketplace + plugin repository structure; users add it as a marketplace from the GitHub URL and install it in one additional command. Hook logic is written in TypeScript, bundled to standalone Node.js files, and committed to `dist/` so no build step is required on install.

**Scale/Scope**: 2 hooks, ~150 lines of source logic, ~10 unit tests per hook

**Structure Decision**: Single-project layout. No backend/frontend split needed — this is a pure plugin with hook scripts and no web interface. The `src/lib/` vs `src/hooks/` split separates testable pure logic from I/O entry points, making unit testing straightforward without mocking stdin/stdout.
